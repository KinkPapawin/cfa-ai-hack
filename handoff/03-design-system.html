Thesis Ledger · Design System v0.1 — Pre-Build Foundation

ระบบออกแบบของผลิตภัณฑ์

รากฐานงานออกแบบที่ตัดสินใจก่อนเขียนหน้าจอ — token, ระบบตัวอักษร, และ component ที่คุมให้ทั้ง 4 หน้าจอเป็นภาษาเดียวกัน นี่คือสิ่งที่แยก "ผลิตภัณฑ์มืออาชีพ" ออกจากงานที่ดูสุ่มเลือกทีละหน้า

Editorial-Institutional Light-first · dark-ready Next.js serif + sans + mono สลับธีมได้ที่ปุ่มมุมขวาบน

01 · Design Principles

หลักการออกแบบ — เฉพาะสำหรับเครื่องมือเฝ้า thesis

หลักการพวกนี้ไม่ใช่คำสวยหรู แต่เป็นข้อบังคับที่ทุก component ต้องเคารพ — มาจากธรรมชาติของผลิตภัณฑ์ที่ "false alarm เยอะ = ความน่าเชื่อถือตาย"

P1

เงียบโดยปริยาย

หน้าจอต้องนิ่งเมื่อไม่มีอะไรเปลี่ยน แถวที่ INTACT พับเก็บ ไม่แย่งความสนใจ — ความเร่งด่วนสงวนไว้ให้ 🔴 จริง ๆ เท่านั้น

P2

สถานะอ่านได้ใน 1 วินาที

ทุก verdict เข้ารหัส 3 ชั้น — รูปทรง + สี + คำ — ไม่เคยใช้สีอย่างเดียว คนตาบอดสีต้องแยก 🔴 ออกจาก 🟢 ได้เสมอ

P3

หลักฐานเป็นพลเมืองชั้นหนึ่ง

ทุกตัวเลข/verdict มีปุ่มสาวกลับไปหน้า/บรรทัดต้นฉบับได้ — ไม่มี claim ไหนลอย ๆ โดยไม่มี provenance

P4

ความหนาแน่นระดับมืออาชีพ

PM สแกน ไม่ได้ถูกจูงมือ — ตัวเลขใช้ตาราง (tabular numerals) เรียงชิด ข้อมูลแน่นแต่ไม่รก อ่านเหมือน research desk

02 · Color Tokens

สี — เลือกอย่างตั้งใจ ไม่ใช่ค่า default

พื้นหลังเป็นโทนกระดาษอุ่น (ไม่ใช่เทากลาง ๆ) accent เป็นน้ำเงินหมึกสถาบัน ใช้เท่าที่จำเป็น ส่วนสีสถานะ 4 ตัวเป็น semantic status ที่สงวนไว้ ห้ามเอาไปใช้เป็นสีตกแต่งอื่น

✓ VALIDATED สีสถานะผ่านการตรวจด้วยสคริปต์จริง (ไม่ใช่กะด้วยตา) — light mode ผ่านทุกเช็ค: lightness band, chroma floor, CVD separation, normal-vision floor ≥15, contrast ≥3:1 · ส่วน dark mode red↔amber อยู่ที่ ΔE ~14 ซึ่งเป็นข้อยกเว้นที่อนุญาตของ status color เพราะมีรูปทรง+คำกำกับเสมอ ไม่ได้พึ่งสีอย่างเดียว

06 · Implementation — Next.js

Token เชื่อมเข้าโค้ดจริงอย่างไร

Design token ทั้งหมดเป็น CSS custom properties → ใช้ได้ทั้งใน globals.css และ map เข้า Tailwind theme ได้ตรง ๆ ส่วนฟอนต์จริงควรใช้ next/font โหลด serif ที่มีลิขสิทธิ์ (เช่น Source Serif / Newsreader) แทน system stack เพื่อความเป็นเอกลักษณ์

globals.css

:root {
  --paper: #F4F2EC;
  --surface: #FCFBF8;
  --ink: #1C1D19;
  --accent: #2B3D5B;
  --intact: #2E7D52;
  --risk:   #C08A1E;
  --broken: #A8322A;
  --stale:  #6B7169;
}
[data-theme="dark"] { /* … */ }

ธีมสลับด้วย data-theme บน <html> — SSR-safe ไม่มี flash

tailwind.config.ts

theme: { extend: {
  colors: {
    paper: "var(--paper)",
    ink: "var(--ink)",
    accent: "var(--accent)",
    status: {
      intact: "var(--intact)",
      risk:   "var(--risk)",
      broken: "var(--broken)",
      stale:  "var(--stale)",
    },
  },
  fontFamily: {
    serif: ["var(--font-serif)"],
    mono:  ["var(--font-mono)"],
  },
}}

Component อ่าน token ผ่าน class — เปลี่ยนธีมทีเดียว กระทบทั้งแอป
