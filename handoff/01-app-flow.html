Exhibit C

3-Day Build Timeline — ใครทำอะไร วันไหน

แมปตรงกับ stage ใน Exhibit A/B — Day 1 หนักสุดเพราะต้อง source ข้อมูลจริงคู่กับเขียน schema

DAY 1 — Foundation

Schema + Data Sourcing

  • รัน B1 falsifiability test ก่อนเริ่มโค้ด
  • คนที่ 1
  • schema + seed DB + entity map + หา public data MWG
  • คนที่ 2
  • หา public data CPALL + เขียน mock thesis note + เริ่ม extraction prompt
DoD: DB seed ครบ 2 บริษัท, verdict คำนวณได้ ≥1 ตัวจากข้อมูลจริง
DAY 2 — Core Loop

Pipeline + Hero Screen

  • คนที่ 1
  • Drop & Trace ต่อ pipeline จริง (scripted/replay)
  • คนที่ 2
  • Adjudicator (comparator + adversarial บน MWG) + traffic-light rule + freshness nudge
DoD: MWG money-shot ถูกต้อง, CPALL โชว์ verdict ตัดกัน
DAY 3 — Integration

Polish + Rehearsal

  • ทั้งคู่
  • Portfolio Board + ปุ่ม cite + Verification Queue (static) + เช็คคำต้องห้าม
  • ซ้อมสคริปต์เต็มรูปแบบ ≥3 รอบ — แก้เฉพาะ show-stopper
ตัดแน่นอน: Slack/LINE, live demo, backtest framework, Thesis Card

Key

🟢 INTACT
ค่าจริงผ่าน threshold + ≥2 แหล่งอิสระ ไม่มี evidence ค้าน
🟡 AT_RISK
โมเดลเห็นไม่ตรงกัน หรืออยู่ในโซน ±20% ของ threshold
🔴 BROKEN
evidence มี page/span ขัด threshold ชัดเจน
⚪ STALE
ไม่มี evidence ใหม่ภายใน decay window (100/30 วัน)
Grounding บังคับ (page/span) Abstention → NO_DATA Human veto ทุก decision Tier policy (licensed ≠ ส่ง LLM) as_of ≠ knowledge_date

ยังไม่ปิด — ต้องทำก่อน/ระหว่าง Day 1

  1. B1 — ยังไม่ได้ทดสอบว่า research note จริงมี numeric criteria พอ falsify ได้ไหม (ทำ manual test 30–60 นาทีก่อนล็อกงาน Day 1)
  2. D2 — ยังไม่มีเจ้าภาพยืนยันว่า MWG money-shot ไม่ใช่ hindsight-fitted (ต้องมีคนรับผิดชอบ + กำหนดเวลา)
